## 1.前言

学习ui组件的时候，因为页面只负责ui部分，不涉及到业务逻辑，所以引申出无状态组件的使用

## 2.无状态组件

无状态组件其实就是函数组件，其从父组件传入props,不方便使用方法，但是如果只是使用ui组件这种的，完全是够了的，并且无状态组件的解析会更加简洁

```jsx
const ToDoList=(props)=>{
    return (
        <div value=props.value></div>
    )
}
```

## 优势：

1.无状态组件的性能会比较高

2.普通组件会有更多的生命周期

3.如果一个组件只有render函数，最好还是使用无状态组件

4.无状态组件也是可以使用简单的逻辑的
